### 修复 Neon fetch 缓存 bug（重要！影响全项目）
- **现象**: migration_007 更新数据后，`/api/shop/cursors` 仍返回旧的 6 条；直接查 DB 返回 20 条。
- **诊断**: route 里加 directCount 对比——`getAllCursorEffects length: 6 | direct count: 20`。同一 sql 对象，旧 SQL 字符串的 fetch 结果被 Next.js 缓存，新 SQL 字符串返回实时数据。
- **根因**: @neondatabase/serverless 底层用全局 fetch 连接 Neon HTTP API，Next.js 14 的 patched fetch 缓存了相同 SQL 的响应。`fetchConnectionCache: false` 只禁用连接复用，不禁用响应缓存。
- **修复**: `lib/db.ts` 的 sql 初始化改为 `neon(url, { fetchOptions: { cache: 'no-store' as RequestCache } })`。需重启 dev server 生效（lib/ 不被 HMR 重载）。
- **影响**: 修复了全项目潜在的查询缓存过期问题（frames 之前"正常"只是因为数据没变）。

## 修复鼠标效果商城闪烁 BUG

### 现象
鼠标效果商城页（/shop → 鼠标效果 tab）预览一闪一闪；全局跟随精灵也有跳动。

### 根因（核心）：SpriteCanvas Image.onload 生命周期泄漏
- Next.js 14 默认开启 React StrictMode，dev 下 useEffect 执行两次（mount→unmount→mount）。
- `SpriteCanvas.tsx` 的 useEffect 创建 `new Image()` 设 onload 后赋 src，但 cleanup 只 `clearInterval(timer)`——此时 timer 还是 0（onload 未触发），且旧 Image 的 onload 未取消。
- 结果：旧 Image 加载完后仍触发 onload，启动一个永不清理的 setInterval，与新 effect 的 setInterval 同时绘制同一 canvas → 闪烁。商城 ~20 个预览放大问题。

### 修复
1. **SpriteCanvas.tsx**（核心）：加 `cancelled` 标志；先设 `img.onload` 再设 `img.src`（防止缓存命中时 onload 先于回调赋值触发）；cleanup 中 `cancelled = true; clearInterval(timer); img.onload = null`；onload 开头 `if (cancelled) return`。
2. **CursorShop.tsx**：移除 `visibilitychange` 重新 fetch 监听（切标签页回来触发整列表重渲染放大闪烁；purchase/equip 成功后已手动 fetch，无需自动刷新）。
3. **CursorFollower.tsx**：移除 style 中的 transform（React 重渲染会用 current.current 快照覆盖 RAF 平滑值）；mount 后立即 `apply(current, current)` 定位避免首帧停留在左上角，RAF 独占 transform 控制。

### 教训
React 中用 `new Image()` 异步加载时，必须在 cleanup 中取消 onload 并设 cancelled 标志，否则 StrictMode 双执行或任何 effect 重跑都会导致旧回调泄漏 + 多 interval 叠加。先设 onload 再设 src 是基本规范。

## 二次修复鼠标效果闪烁（真正的根因）

### 现象
第一次修复（Image.onload 生命周期）后，用户反馈仍然闪烁——商城预览和全局跟随精灵都在闪。

### 真正的根因：SpriteCanvas fixedMode 每帧包围盒缩放不一致
- `fixedMode`（传了 width/height 的模式，鼠标跟随 + 商城预览都用）下，`draw()` 每帧取该帧的非透明像素包围盒 `box = boxes[idx]`，用 `scale = Math.min(cw / box.w, ch / box.h)` 计算缩放。
- **问题**：每帧角色姿态不同 → 包围盒宽高不同 → scale 不同 → 绘制位置 `(cw-dw)/2` 也不同 → 角色每帧大小和位置都在跳 → 闪烁。
- 非固定模式（PetSprite）不受影响，因为 canvas 尺寸取所有帧包围盒的最大值，每帧用各自包围盒居中是预期行为。

### 修复
1. **SpriteCanvas.tsx**（核心）：
   - `fixedMode` 跳过包围盒计算（72 帧 × getImageData 的开销也省了，加载更快）
   - 用帧尺寸做统一缩放：`fixedScale = Math.min(cw / frameW, ch / frameH)`，预计算 `fixedDx/Dy/Dw/Dh` 所有帧共用
   - draw() 中 fixedMode 分支直接用预计算值，每帧只是源区域不同，目标区域恒定 → 无闪烁
2. **CursorPreview.tsx**：加 `React.memo` + `useMemo(sprite)`，防止 CursorShop 状态变化（busy/msg）时所有预览无谓重渲染
3. **CursorFollower.tsx**：sprite 对象加 `useMemo`，防止 motion 变化时创建新对象

### 教训
精灵图动画的每帧包围盒（非透明区域）天然不同。如果在固定尺寸画布上用每帧包围盒做缩放，角色会每帧跳。正确做法是统一缩放（用帧尺寸或所有帧包围盒的并集），每帧只是裁剪源区域不同。
